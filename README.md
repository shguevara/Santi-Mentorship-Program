# Mentorship Program Overview

## Purpose

This mentorship program is designed to help you develop discipline, strong fundamentals, and a professional mindset for software engineering.

The focus is not on rushing through courses. The focus is on learning how engineers actually train, build, and improve their work.

You will work through a structured system where progress is earned by completing work to a clear standard.

🎯 The point is simple: become the kind of person who can reliably deliver high-quality work, even when it gets hard.

---

## Why This Program Exists

Many smart, self-taught learners make the same mistake: they jump between topics and tools, start many things, and finish very little. That creates shallow confidence and weak professional habits.

This program exists to build the opposite:

- Completion discipline
- Comfort with feedback and rewrites
- Strong fundamentals
- Clear standards
- A professional identity built through delivered work

🔥 If you can learn to finish to a standard, you will already be ahead of most people.

---

## Core Principles

1. Finish what you start before beginning something new.
2. Working code is the baseline, not the goal.
3. Clean, understandable code is the real standard.
4. Feedback and rewrites are normal parts of improvement.
5. Progress is measured by completed work, not hours spent.
6. Each level must be completed before the next one unlocks.
7. No new technology until the current level is completed.

---

## How the Program Works

The program is organized into **levels**. Each level follows the same structure:

1. Learning block (course or study material)
2. Product challenge
3. Sprint-based development
4. Demo of your work
5. Code review
6. Improvements and rewrites
7. Retrospective reflection
8. Project shipped to GitHub

🔓 Only after a level is completed to standard will the next level unlock.

✅ Each level ends with a finished project, a demo, and a reflection.

---

## Level Map

This is the progression path. Think of it as a ladder, one rung at a time 🪜:

1. **Level 1: Python Foundations**
2. **Level 2: Frontend Foundations**
3. **Level 3: Backend Foundations**
4. **Level 4: Full Stack Project**
5. **Level 5: Team Simulation Project (with Matheo)**

Each level ends with a finished project, a demo, and a reflection.

---

## Cadence

- Bi-weekly sprint rhythm (2-week sprints)
- One demo and code review per sprint
- One short written retrospective per sprint
- One level completion review when a level is finished

---

## Communication Protocol

When you are blocked, communicate like a professional:

- What you are trying to do
- What you expected to happen
- What happened instead (include the error)
- What you already tried
- What you think the next options are

🧠 Avoid messages like: "It doesn’t work." Bring evidence and hypotheses.

---

## Work Standards

A feature is not considered complete just because it runs.

Standards include:

- Clean naming and readable structure
- Minimal duplication
- Clear error handling
- Meaningful commits
- Ability to explain decisions and tradeoffs

---

## Review Process

After each sprint:

- You demo what you built
- We do a code review
- You apply feedback
- Rewrites or a quality pass may be requested

🎬 Each sprint ends with a demo and review.

Rewrites are normal and expected.

---

## Learning Discipline

Courses support the work, but they are not the work.

Progress is proven through:

- Finished features
- Clean code
- Delivered demos
- Shipped projects
- Written reflections

---

## Progression System

- Advancement is based on standards, not time.
- New technologies are unlocked only after the current level is completed.
- If standards are not met, we continue the current level until they are.

---

## Portfolio Development

Every level produces a portfolio package:

- A finished project
- A GitHub repository with clear commit history
- A README describing what it does and how it is structured
- A short reflection documenting what you learned and what you improved

---

## Level 1: Python Foundations

Goal: Build strong programming discipline and learn how to complete technical work to a professional standard.

You will:

- Complete a Python fundamentals learning block
- Learn Git basics
- Build a CLI application based on a small backlog of user stories

During this level you will work in bi-weekly sprints (2-week sprints) using a simple board:

- To Do
- Doing
- Done

🎬 Each sprint ends with a demo and review.

---

## Product Challenge

After the learning block, you will build a small project using a backlog of user stories.

Example:

User Story:

As a user, I can create a task with a title and description so I can track work I need to do.

Acceptance criteria define exactly what "done" means.

---

## AI Usage

AI tools can be powerful but they must be used correctly.

Rules:

- First attempt should be done independently
- AI can help critique or suggest improvements
- AI should not replace understanding

🧩 The goal is to strengthen your thinking, not bypass it.

---

## Moving to the Next Level

Promotion to the next level depends on:

- Quality of the project
- Ability to explain your work
- Response to feedback
- Completion discipline
- GitHub repo quality (commits, README, structure)

If standards are met, the next level unlocks.

If not, the current level continues until they are.

---

## Professional Behavior Expectations

This mentorship expects professional habits:

- Show up prepared
- Respect time and commitments
- Communicate clearly and early when blocked
- Accept critique without defensiveness
- Take ownership of quality

---

## 30-Day Review

At day 30 we will do a review and continuation decision.

We will evaluate:

- Consistency of effort
- Quality of delivered work
- Response to feedback and rewrites
- Evidence of stronger discipline and focus
- Whether the structure is working for you

🤝 If we both agree it is working, we continue. If not, we pause or end cleanly.

---

## Mentorship Working Agreement

This mentorship works only if both sides are committed. The following expectations define how we will work together.

### My Commitment (Mentor)

I commit to:

- Providing structured guidance and direction
- Reviewing your work and giving honest feedback
- Helping you understand how real engineering work happens
- Challenging you to improve your thinking and coding discipline
- Sharing experience from real projects and teams

My goal is to help you develop strong habits and a professional mindset early in your journey.

---

### Your Commitment (Mentee)

You commit to:

- Showing up prepared for demos and discussions
- Finishing what you start
- Accepting feedback without taking it personally
- Rewriting or improving work when requested
- Communicating clearly when you are blocked
- Maintaining consistent effort during the program

Progress in the program depends on your effort and follow-through.

---

### What This Program Is

This program is:

- A structured mentorship
- A place to develop discipline
- A safe environment to receive real feedback
- A training ground for professional engineering habits

The goal is growth through practice, iteration, and reflection.

📝 Reflection is important for building strong habits.

---

### What This Program Is Not

This program is not:

- A casual coding club
- A place to jump between many technologies
- A shortcut to becoming a developer quickly
- A place where "good enough" work passes

The emphasis is on depth, completion, and craftsmanship.

---

### The Shared Goal

The purpose of this mentorship is simple:

To help you begin forming the habits, thinking patterns, and work ethic of a professional engineer.

🚀 If you show up consistently and do the work, you will build momentum that compounds.

If both sides uphold this agreement, the program will produce real progress.
